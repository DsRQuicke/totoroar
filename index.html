After the resources have loaded and when WebXR and its AR module are available, you can enter the AR experience by pressing the glowing logo. The first time you do this, the browser will ask for permission to start an AR session. Once you're in AR mode, and a sufficiently stable 3D map of the environment is available (see Move device below), a recticle will be shown. When you then touch the screen, the model will be placed at the location of the recticle. The recticle will now be hidden until the model is removed again.

The following UI elements are available:

Exit AR: Press this to exit the AR experience.

Remove model: Removes the model. The recticle will be activated again, so the model can be placed at another location.
This button is only enabled when the model has been placed.

Toggle scale: Toggles the model between normal and full size.
This button is only enabled when the model has been placed.

Umbrella: Take out or put away umbrella.
This button is only enabled when the model has been placed.

Whistle: Start or stop whistling.
This button is only enabled when the model has been placed.

Move device: This icon indicates that the user should move their device around to help build the underlying environment map.
To be able to perform AR hit tests, the underlying AR framework first needs to understand the environment near the user. It does this by progressively building up a 3D map while the device is moved through the environment. To expedite this process, the icon is shown to encourage the user to move their device around. A well lit environment additionally improves this process.
Note that no recticle will be shown until enough environment information is available. Also note that if you point or move your device to a new part of the environment, then this icon might be shown again while the environment map is being expanded.

This app uses the AR module of the WebXR standard. Since that module is fairly new, your browser might not yet support it. Or you might have to enable it explicitly. Chrome v81 and up should support it out of the box.
